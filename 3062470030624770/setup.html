🌅 Dashboard Setup

Get your morning dashboard running in a few steps. Come back here any time to update your configuration.

Todoist — task list
not tested
›
Your tasks are pulled live from Todoist every 60 seconds. You need to give the dashboard your API token via a Vercel environment variable.
  1. Go to Todoist → Settings → Integrations → Developer
  2. Copy your API token (a long string of letters and numbers)
  3. Open the Vercel dashboard → find the MathAns project → Settings → Environment Variables
  4. Add a new variable:
KeyValue
TODOIST_TOKEN Paste the API token you copied from Todoist
Make sure you set the environment for Production (and Preview if you want). Then trigger a new deployment (or just wait for the next deploy).

2
Google Calendar
not tested
›
The dashboard reads your calendar via a private iCal URL. This is a secret link Google gives you — no OAuth needed.
  1. Open Google Calendar on desktop
  2. In the left sidebar, hover over the calendar you want to show → click the ⋮ menu → Settings and sharing
  3. Scroll down to "Secret address in iCal format" and copy the URL (starts with https://calendar.google.com/calendar/ical/…)
  4. In Vercel → MathAns → Settings → Environment Variables, add:
KeyValue
GCAL_ICAL_URL The full iCal secret URL from Google Calendar
Tip: You can add multiple calendars by combining them. Just configure the primary one first and test it.

5
Deploy & verify
›
After editing index.html with your config and committing + pushing to GitHub, Vercel auto-deploys. Here's how to verify everything is working.
  1. Push your index.html changes (with BUS_STOPS and JOURNEYS filled in) to the main branch on GitHub
  2. Wait ~30 seconds for Vercel to deploy (you'll see a green tick in your GitHub commit)
  3. Open mathans.app/3062470030624770 in the Silk browser on your Echo Show 8
  4. Check each panel loads — Tasks (left), Calendar (centre), Bus ETAs (right)
  5. Tap the Journey tab to verify the stop timeline loads
Troubleshooting:
· Tasks empty or error → check TODOIST_TOKEN is set in Vercel env vars
· Calendar empty → check GCAL_ICAL_URL and that it's the iCal (not HTML) URL
· Bus shows nothing → confirm your stop IDs are valid (use the finder in Step 3)
· Journey shows "No service" → the bus isn't running right now, try again later
